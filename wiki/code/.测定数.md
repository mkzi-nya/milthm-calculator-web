# 测定数方法
## 获取chart id
- 在[Rainseek仓库](https://github.com/MilthmLang/RainSeek.Dataset.Milthm)获取

- 在[查分器仓库](https://github.com/mkzi-nya/milthm-calculator-web/blob/main/js/constant.js)获取（不能及时更新）

- 在apk解包后的Assets/Beatmaps/Runtime/下解压获取（不保证准确）

## 测量方式

修改saves.db内的kv内的json数据，将其所有分数改为0（其他不用变）

然后修改其中一项成绩的id为需计算的曲目id，将成绩改为990000（单曲rlt为定数+1.4）

然后进入游戏，记录当前reality，随后再次修改为989999并再次记录

假设两次记录的值分别为0.67和0.67，则定数为0.67*20-1.4=12

假设值为0.67和0.66，则定数为0.67*20-1.4-0.1=11.9
